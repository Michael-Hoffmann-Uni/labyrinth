\section{Zielbestimmung\label{sec:Zielbestimmung}}
\subsection{MUSS-Kriterien\label{sec:MUSSKriterien}}
\begin{enumerate}
	\item Die Regeln des Spiels "`Das verrückte Labyrinth"' werden gemäß der offiziellen Anleitung\footnote{\url{https://www.ravensburger.de/spielanleitungen/ecm/Spielanleitungen/26446\%20anl\%201637944.pdf?ossl=pds_text_Spielanleitung}} implementiert, wobei nur die Grundvariante die relevante Regelgrundlage darstellt.
	
	In Abweichung zur Anleitung beginnt ein zufälliger Spieler.
	\item Das Spiel wird mit grafischer Benutzerschnittstelle implementiert, sodass der menschliche Spiele ein dem gewohnten Spiel ähnliches Spielfeld steuert.
	\item Je Spielrunde kann maximal ein menschlicher Spieler teilnehmen, die anderen Spieler werden von KIs gesteuert.
	\item Das Spiel ermöglicht einen Simulationsmodus ohne menschliche Mitspieler, in dem zwei bis vier Spieler von (verschiedenen) KIs gesteuert werden. Dieser Modus kann sowohl mit als auch ohne grafische Anzeige des Spielverlaufs ablaufen. Endlos-Spiele\footnote{Treffen hinreichend "`dumme"' KIs aufeinander sind ergebnislose Spielverläufe denkbar -- diese werden durch zeitliches Limit oder andere geeignete Kriterien beendet und nicht gewertet.} werden durch geeignete Kriterien beendet.
	\item Die KIs liegen zur Kompilierzeit vor und steuern über eine geeignete Schnittstelle das Spielgeschehen, wenn der von ihr vertretene Spieler an der Reihe ist. Über die Schnittstelle steht der bisherige Spielverlauf zur Verfügung.
	\item Eine spielfähige KI wird implementiert; die Züge der KI basieren auf dem zum Zeitpunkt des Zuges aktuellen Spielfeldsituation und betrachtet mögliche zukünftige Züge nicht.
	\item Die Spielergebnisse vollständig abgeschlossener Spielrunden werden in eine SQL-Datenbank geschrieben; aus den Einträgen ist ersichtlich
			\begin{enumerate}
				\item welche Spieler/KIs an dem Spiel teilgenommen haben und
				\item welcher Spieler siegreich war.
			\end{enumerate}
	\item Aus der Datenbank können statistische Informationen gewonnen werden, welche grafisch aufbereitet und menschlich lesbar angezeigt werden.
	\item Die Umsetzung des Projekts erfolgt mit Java, der Code wird per Git verwaltet.
\end{enumerate}

\subsection{KANN-Kriterien\label{sec:KANNKriterien}}
\begin{enumerate}
	\item Der vollständige Spielverlauf wird in der Datenbank gespeichert.
	\item Die grafische Anzeige des Spiels erfolgt mit fotorealistischen Darstellungen.
	\item Menschliche Spieler an einem PC können namentlich unterschieden werden.
	\item Spielvarianten, die über Menü-Einstellungen ausgewählt werden können:
		\begin{enumerate}
			\item Die Spielvariante für Kinder nach offizieller Anleitung wird implementiert.
			\item Die Variante des "`Labyrinth der Meister"' wird implementiert.
			\item Trainingsmodus: der menschliche Spieler erhält optische Unterstützung für seine möglichen Züge.
		\end{enumerate}
	\item Es werden verschiedene KIs implementiert.
\end{enumerate}


\subsection{Abgrenzung\label{sec:Abgrenzung}}
\begin{enumerate}
	\item Das Spiel bietet keinen menschlich steuerbaren Mehrspieler-Modus, weder über Netzwerk noch am gleichen Endgerät.
	\item Abgebrochene Spielrunden können weder gespeichert noch fortgesetzt werden.
\end{enumerate}
